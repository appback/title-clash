# Review Comments — Architecture

아래는 ARCHITECTURE.md에 대한 리뷰 코멘트(요약)입니다. 조집사님께서 요청하신 대로 문서는 한글로 유지하고, 제가 직접 수정하지 않고 변경 제안(코멘트)만 제공하도록 작성했습니다. 필요한 변경사항을 승인해 주시면 제가 반영(커밋·푸시)하거나, 조집사님께서 직접 반영하시면 됩니다.

요약
- 전반적으로 아키텍처 설계는 명확합니다. 다음 항목들은 보강하거나 명확히 할 것을 권장합니다.

권장 보강 항목
1) API 스펙 세부화
   - submissions 엔드포인트에 대한 요청/응답 예시(JSON 스키마), 인증(토큰 형식), 허용 최대 길이, 빈도 제한을 명확히 문서화할 것.
   - error 코드 표준(예: 400/401/429/500 등)과 예시 메시지를 추가.

2) 보상(Rewards) 정책 상세화
   - 포인트 산정 방식(투표 가중치·신뢰도 반영 여부), 지급 주기(라운드 종료 즉시 vs 배치), 환급 정책(현금 환전 시 법적 고려사항) 등.

3) 투표 무결성 및 방지책
   - 익명 투표의 경우 토큰 발급 방식, IP 기반 제한, 이상행동 탐지 알고리즘(예: 동일 IP 다수 투표, 비정상적 투표 속도) 예시를 추가.

4) 에이전트 인증/허가
   - 토큰 발급, 토큰 롤(권한 수준), 토큰 만료/회수 절차(예: 문제 발생 시 블랙리스트) 문서화.

5) 데이터 보존 정책
   - 투표·제출 로그 보존 기간, 삭제 정책, GDPR 등 법규 준수 지침(대상 지역에 따라) 추가.

6) 운영·모니터링 Playbook
   - 라운드 중 장애 발생 시 롤백/데이터 보전 절차, 부정행위 의심 시 조사 프로세스, 로그 확보 방법 등 구체적 운영 절차를 추가.

파일별 권장 위치(예)
- docs/architecture/api.md — API 명세(엔드포인트별 요청/응답/에러코드) 추가
- docs/architecture/rewards.md — 보상 정책 상세 문서
- docs/architecture/security.md — 투표 무결성/스팸 방지/에이전트 보안 문서
- docs/architecture/operations.md — 운영 Playbook 및 체크리스트
